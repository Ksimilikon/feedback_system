Некоторые функции возвращают целочисленное значение о успешности выполнения функции
"1" - функция выполненна успешност
"-1" - функция не выполненна

Если у вас не будет достаточно прав доступа, то вас перенесёт на страницу ошибки
Ошибки хранятся в массиве SESSION: ключ error - 1а ошибка в классе MyError, ключ errors - массив ошибок, которые являются классом MyError
Ошибки аутентификации - errors_auth

Коды и описание ошибок хранятся в папке Errors в файлах txt. Используются для реализации класса Error

Коды ошибок:
1 - пользователя не существует 
2 - Неверный пароль
3 - Вход не осуществлён
4 - недопустимые символы
5 - Короткий пароль, минимум 6 символов
6 - Поле должно содержать минимум 2 символа
7 - Недостаточно прав
8 - Ошибка запроса
9 - недостаточно данных

Коды ошибок авторизации/регистрации:
1110 - заполните поле с электронной почтой
1111 - заполните поле с паролем
1112 - заполните поле с именем
1113 - заполните поле с фамилией
1114 - Повторите пароль
1115 - Пароль слишком Короткий
1116 - Пароль слишком длинный
1117 - Пароли не совпадают 
1118 - Пользователя с такой электронной почтой не существует
1119 - Пользоваетель с такой электронной почтой уже существует

На главной странице показывают информации о пользователе, которую можно изменить
Страницы с запросами содержат фильтры; запросы осуществляются через js ajax

Каждое сообщение об ошибке - чат-форум 
Пользователи могут искать уже решенные вопросы и найти ответ сами по словам в сообщении

Роли:(чем больше индекс, тем выше доступ)
5 - user: обычный пользователь; может отправлят сообщения с ошибками, просматривать свои сообщения и просматривать ответ на сообщение
10 - moder: решает заданные вопросы
20 - admin: может менять пользователей, редактировать сообщения

Классы хранятся в папке php/class
	Auth: авторизация и проверка доступа пользователя к странице
	MyError: класс ошибки, для централизованного управления текстом ошибки
	Debug: предназначен для вывода информации об ошибках во время отладки
	GuardHandler: защищает обработчики от несанкционированного и непредусмотренного доступа
		к скрипту
	PrepareForSQL: содержит методы для проверки и подготовки строк для записи в базу данных
	Reg: регистрация
	Location и LocationWhenError: перемещают пользователя на другую страницу сайта
Классы запросы php/request
	User: получает все данные о пользователе и позволяет их получить в удобной форме
	RequestUpdate: позволяет быстро обновить данные в таблеце базы данных

Функция php/class/func_checkArrKey.php проверяет существование ключа в массиве и наличие значения
	в массиве в функцией empty(). Если ключ есть и значение не пустое, то возвращает 1,
	иначе если нет ключа -1, если значение пустое -2
